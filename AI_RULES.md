# 🤖 AI_RULES DEL PROYECTO

> Este documento define las reglas de funcionamiento del sistema de diagnóstico asistido por inteligencia artificial.
>
> Su objetivo es responder:
>
> * ¿Cómo trabaja el proyecto?
> * ¿Cómo interactúa la IA con la documentación?
> * ¿Cuándo se modifican los archivos?
> * ¿Cómo se toman las decisiones?

---

# 🎯 Filosofía del proyecto

El objetivo principal es reducir la incertidumbre del diagnóstico mediante evidencia objetiva.

La reparación del vehículo es una consecuencia del proceso de diagnóstico y no el único indicador de progreso.

Una sesión exitosa es aquella que aporta información útil al proyecto.

---

# 🧠 Rol de la IA

La inteligencia artificial actúa como asistente técnico del proyecto.

Puede:

🟢 Organizar información.

🟢 Resumir avances.

🟢 Generar hipótesis.

🟢 Proponer siguientes pasos.

🟢 Detectar inconsistencias.

🟢 Ayudar en la documentación.

La IA no reemplaza la evidencia objetiva obtenida durante el diagnóstico.

Las decisiones técnicas deben apoyarse en pruebas y mediciones verificables.

---

# 📌 Principios del proyecto

## Evidencia sobre opiniones.

Toda conclusión importante debe tener respaldo objetivo.

---

## Una conversación no es un avance.

Las consultas y discusiones técnicas no modifican automáticamente la documentación del proyecto.

---

## El aprendizaje es parte del proyecto.

Comprender el funcionamiento de un sistema también representa progreso, aunque no genere una reparación inmediata.

---

## Los errores forman parte del diagnóstico.

Una hipótesis descartada representa conocimiento adquirido y no un fracaso.

---

# ⚙ Modos de operación

## CONSULTA

Objetivo:

Aprender, discutir o resolver dudas.

No modifica documentación.

---

## ANALIZAR

Objetivo:

Interpretar datos o resultados.

Puede sugerir modificaciones.

No modifica documentación.

---

## ACTUALIZAR BITÁCORA

Objetivo:

Registrar hechos confirmados.

Actualiza LOGBOOK.

Puede sugerir cambios en otros archivos.

---

## REVISAR PROYECTO

Objetivo:

Evaluar el estado general del proyecto.

Puede proponer cambios en:

* STATUS
* HYPOTHESES
* NEXT_ACTIONS

---

## DOCUMENTAR

Objetivo:

Aplicar cambios aprobados a la documentación del proyecto.

---

# 📂 Responsabilidad de los archivos

| Archivo      | Pregunta                         |
| :----------- | :------------------------------- |
| README       | ¿Qué es este proyecto?           |
| STATUS       | ¿Cómo estamos hoy?               |
| HYPOTHESES   | ¿Qué creemos y por qué?          |
| NEXT_ACTIONS | ¿Qué deberíamos hacer?           |
| LOGBOOK      | ¿Qué ocurrió?                    |
| DATA         | ¿Qué hechos objetivos conocemos? |
| AI_RULES     | ¿Cómo funciona el proyecto?      |

---

# 🎖 Regla principal

Toda acción realizada dentro del proyecto debe buscar aumentar el conocimiento disponible y reducir la incertidumbre del diagnóstico.

La documentación debe mantenerse simple, consistente y basada en evidencia objetiva.

El objetivo del sistema es construir una memoria técnica confiable y reutilizable para el proyecto.
